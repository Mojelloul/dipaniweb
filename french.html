Nos choix technologiques

Un stack optimisé pour Dipani

Pourquoi React Native ?

Framework open-source développé par Facebook pour créer des applications mobiles cross-platform (iOS et Android) avec une seule base de code.

Avantages clés :

  • Développement rapide & économique → Un seul code pour deux plateformes
  • Performance fluide → Proche d'une application native
  • Maintenance simplifiée → Moins d'efforts pour les bugs et nouvelles fonctionnalités
  • Communauté et plugins riches → Large écosystème d'outils
  • Évolutivité → Facile à scaler avec des mises à jour fréquentes

Pourquoi ne pas utiliser Flutter ou une application native ?

  • Flutter → Excellente alternative, mais React Native a plus de développeurs et une meilleure compatibilité
  • Applications natives (Swift/Kotlin) → Plus coûteuses et longues à développer

Pourquoi Supabase ?

Alternative open-source à Firebase, offrant une base de données PostgreSQL avec API automatique et une gestion d'authentification intégrée.

Avantages clés :

  • Base de données évolutive → PostgreSQL, robuste et performant
  • Backend en No-Code/Low-Code → API générées automatiquement
  • Authentification sécurisée → Gestion des utilisateurs avec OAuth, email/password
  • Stockage de fichiers intégré → Idéal pour gérer les images et vidéos
  • Open-source & Moins cher que Firebase → Contrôle des données et coûts réduits

Pourquoi ne pas utiliser Firebase ?

  • Firebase peut devenir très coûteux avec le volume d'utilisateurs
  • Supabase permet une meilleure scalabilité à long terme
  • Pas de dépendance à Google
